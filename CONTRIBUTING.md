# Contributing

Thanks for considering a contribution.

## Workflow

1. Open or claim a Beads issue with `bd`.
2. Make one logical change per commit.
3. Add or update fixture-based tests for parser/classifier changes.
4. Run verification locally before opening a PR.
5. Use conventional commits.

## Scope

Good first contributions are rule-table improvements, parser fixtures, and report wording. Avoid adding process-killing, telemetry, or privileged actions without an issue and design discussion.
